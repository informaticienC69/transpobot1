💻Frontend & Chat UI : HTML/JS Intégré
SÉLECTIONNÉ

Garantie d'une interface robuste pour la gestion KPI et l'intégration Chatbot.

  • Vanilla JS (sans React/Vue) : Nous avons opté pour du HTML/JS/CSS natif pour réduire significativement le surcoût de déploiement et maximiser la rapidité de création des Dashboards interactifs (Chart.js).
  • Chatbot Web Intégré : Plutôt que de déléguer à Telegram, le chat Text-to-SQL est intégré directement dans le Dashboard (Panneau furtif Cmd+K) pour qu'un agent puisse poser des questions tout en analysant ses données CRUD.
⚡Backend : FastAPI (Python)
SÉLECTIONNÉ

FastAPI a été privilégié catégoriquement face à d'autres alternatives comme Node.js.

  • Mode Asynchrone : Absolument nécessaire pour ne pas bloquer les requêtes MySQL pendant que l'API interroge le LLM.
  • Écosystème IA : Python offre les bibliothèques d'orchestration IA les plus matures et performantes du marché.
🐬Base de données : MySQL 8.x
SÉLECTIONNÉ

Implémentation stricte d'un modèle relationnel complet et intègre.

  • Schéma enrichi : Intégration stricte des tables. Ajout de contraintes métier poussées (gestion anti-fatigue 8H, maintenance prédictive).
  • Text-to-SQL sécurisé : Seuls les appels d'extraction ('SELECT') sont tolérés via un routage strict côté Backend. Si l'IA génère un 'DELETE', l'API le rejette.
🤖Intelligence IA : Groq Cloud API
SÉLECTIONNÉ

Service LLM ultra-rapide basé sur le cloud, privilégié face à OpenAI ou à des modèles locaux.

  • Modèle LLaMA-3.3-70b-versatile : Modèle open-source de Meta, hébergé sur Groq. Génère le SQL (text-to-SQL) ET la réponse en langage naturel. Température 0.0 pour la génération SQL (déterministe) et 0.4 pour la réponse narrative.
  • Budget & Gratuité : Groq propose une API gratuite, là où OpenAI impose des coûts récurrents par token, bloquants à notre échelle.
  • Librairie Python groq : Le code utilise directement le SDK Groq (import Groq). ⚠️ Incompatible API avec le SDK openai — le schéma prompt est injecté en system message.
  • Modèles Locaux (Ollama LLaMA3) écartés : Déployer un LLM lourd exige une puissance GPU qui explose drastiquement les coûts matériels du déploiement.
☁️Déploiement : Render (Backend) + AlwaysData (MySQL)
EN PRODUCTION

Architecture cloud hybride : Backend Python hébergé sur Render, base de données MySQL hébergée sur AlwaysData.

  • Render (Backend FastAPI) : Héberge l'API Python. Déploiement automatique depuis GitHub (CI/CD). HTTPS géré automatiquement. Plan gratuit avec 750h/mois. Redéploiement automatique à chaque git push sur main.
  • AlwaysData (MySQL 8.0) : Héberge la base de données informaticienc_transpobot. Plan gratuit avec 100 Mo. Connexion sécurisée SSL depuis Render. Les credentials sont stockés dans les variables d'environnement Render (jamais dans le code).
  • Serveur VPS classique écarté : Bien qu'un VPS offre le contrôle total, il nécessite des compétences SysAdmin (FireWall, Nginx, Certificats SSL auto-renouvelés) qui détournent du développement. Render gère le HTTPS et le déploiement de manière automatisée (PaaS).
