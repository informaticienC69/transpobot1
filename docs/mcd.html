⚠️ Rappel Merise : Le MCD représente le niveau conceptuel — pas de clés étrangères. L'identifiant est souligné. UTILISATEUR et LOGS_REQUETES sont des entités système (couche auth/IA), sans association avec le modèle métier transport. Cardinalités : 0,1 = zéro ou un · 1,1 = exactement un · 0,N = zéro ou plusieurs · 1,N = un ou plusieurs. Attributs ★ = enrichissements pour optimiser les requêtes IA (Text-to-SQL).

📖 Lecture des 10 associations

# Association Cardinalités Lecture & utilité IA
① VÉHICULE — conduit — CHAUFFEUR (0,N) : (0,1) Un véhicule est attribué à 0 ou 1 chauffeur. Un chauffeur peut avoir conduit 0 ou N véhicules dans le temps.
② VÉHICULE — utilise — TRAJET (0,N) : (1,1) Un trajet utilise exactement 1 véhicule. Un véhicule peut figurer dans 0 ou N trajets.
③ VÉHICULE — subit — MAINTENANCE (0,N) : (1,1) Une maintenance concerne exactement 1 véhicule. Un véhicule peut subir 0 ou N interventions. L'attribut kilometrage_seuil ★ permet à l'IA de répondre : "Quels véhicules nécessitent une maintenance ?"
④ CHAUFFEUR — effectue — TRAJET (0,N) : (1,1) Un trajet est effectué par exactement 1 chauffeur. Jointure INCIDENT→TRAJET→CHAUFFEUR permet : "Quel chauffeur a le plus d'incidents ?"
⑤ TRAJET — planifie — LIGNE (1,1) : (1,N) Un trajet appartient à exactement 1 ligne. Une ligne regroupe 1 ou N trajets.
⑥ LIGNE — applique — TARIF (1,N) : (1,1) Un tarif est propre à 1 ligne. Une ligne définit 1 ou N tarifs (normal / adulte / enfant). Le tarif normal = plein tarif, adulte = réduit adulte, enfant = réduit enfant.
⑦ LIGNE — passe_par — ARRET
+ ordre, temps_estime		 (1,N) : (0,N) Association N:M → table ligne_arrets dans le MLD. temps_estime ★ permet de calculer l'heure d'arrivée à un arrêt précis.
⑧ TRAJET — génère — INCIDENT (0,N) : (1,1) Un incident est lié à exactement 1 trajet. Un trajet peut générer 0 ou N incidents.
⑨ LIGNE — définit — HORAIRE (1,N) : (1,1) Un horaire appartient à 1 ligne. Une ligne a 1 ou N horaires. Permet à l'IA de calculer : retard = heure_reelle − heure_theorique.
⑩ UTILISATEUR — génère — LOGS_REQUETES (1,1) : (0,N) Un log appartient à 1 utilisateur. Un utilisateur génère 0 ou N requêtes IA. Permet l'audit et l'historique du chatbot.
